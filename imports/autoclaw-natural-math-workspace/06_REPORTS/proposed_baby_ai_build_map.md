## Layer Integrations

### 1. Natural Math → Supports Bounded Local Process and Experimental Learning Rules

Natural Math v5 provides:
- **Bounded local generative process:** Integer-space morphogenesis with rigid constraint envelopes (Scale Gates 1–8 validated)
- **Cluster formation:** Documented via oracle fixtures, runners, and results (33 verified evidence items)
- **Scale-gate stability:** Parameter sensitivity, seed sweeps, two-parameter interactions, long-run stability — all verified via CSV/JSON evidence
- **No global planning:** The frozen v5 mechanics are strictly local; hidden global planning is not permitted
- **Role in baby-AI:** Provides the low-level space where patterns emerge locally. The baby-AI does not alter this layer — it reads from it, builds associations on top of it, and may route through it.

**How Natural Math supports learning rules:**
- The integer base provides a well-defined state space with known invariants
- The bounded generative process offers a substrate that can be observed, associated, and scaffolded
- Scale gates define the safety envelope within which experimental learning must operate
- Cluster formation dynamics can be instrumented as a primitive "pattern detector" that higher layers can reference

### 2. Cognitive Basin → Supports Persistent Association, Attractors, Contradiction Scars, Recovery Routes, and Understanding-Like Routing

Cognitive Basin provides:
- **Persistent association:** Memory routing that survives individual experiences (`operational_self_whitepaper_v0_1.md`)
- **Attractors:** Convergent states that stabilize repeated patterns
- **Contradiction scars:** Persistent markers where contradictions were encountered (`contradiction_scars.json`)
- **Recovery routes:** Structured replay-route artifacts for recovering from contradiction (`replay_routes.json`)
- **Narrative continuity:** Self-narrative frame and team-narrative for orientation over time (`self_narrative_frame.json`)
- **Understanding-like routing:** Bounded replay that governs interpretation without collapsing into a single mechanism
- **Team Kernel:** Technical glossary and operating-rules kernel defining the cognitive operating model (`Synaptient_Cognitive_Basin_Team_Kernel_v0.6.md`)
- **Minimal Runtime:** Bounded cycle that frames PERCEPT, ATAL, RIGOR, CIRCUIT, GUARD, and SERA interaction (`Codex_Task_Cognitive_Basin_Core_v0_1_Minimal_Runtime.md`)

**How Cognitive Basin supports baby-AI development:**
- The Basin is the "memory and understanding" layer — it persists what Natural Math generates
- Associations form between Natural Math clusters and Basin-attractors
- Contradiction scars prevent the baby-AI from silently overwriting contradictory experience
- Recovery routes let the system find its way back after failure
- The narrative frame gives continuity across sessions

### 3. Fractalish → Provides PERCEPT, ATAL, RIGOR, CIRCUIT, GUARD, and SERA

The Fractalish stack frames how the operating components interact:

| Component | Role |
|---|---|
| **PERCEPT** | Intake — what enters the system from the environment |
| **ATAL** | Training — staged experience introduction via APTD doctrine |
| **RIGOR** | Constraint — ensures bounded, verifiable process |
| **CIRCUIT** | Flow — the connective tissue between components |
| **GUARD** | Safety — zero-trust intake, contamination handling (Guardian Intake) |
| **SERA** | Resilience — efficiency, stability, recovery metrics |
| **HOLD** | Pause — protected unresolved state; prevents false closure |
| **MCVA** | Verification — minimum cross-verification architecture |
| **AMCVA** | Expanded verification — additional verification dimensions |

**How Fractalish supports baby-AI development:**
- PERCEPT gates what the baby-AI can experience
- ATAL stages when and how training experiences are introduced
- RIGOR ensures experiments stay bounded
- CIRCUIT connects components without collapsing them
- GUARD enforces intake safety
- SERA measures whether the system remains stable
- HOLD prevents premature conclusions

### 4. The Fledgling Baby-AI → Develops Through Controlled Tasks and Separately Tested Extensions

The baby-AI is not a single mechanism. It is assembled from:

1. **Natural Math base** (frozen, non-negotiable)
2. **Cognitive Basin memory** (persistent associations, scars, routes)
3. **Fractalish operating layer** (gate, train, verify, hold)
4. **Separately tested extensions** (v5.x experimental work, activation experiments)

**Developmental constraints:**
- One conceptual mechanism must be tested at a time
- No extension may silently replace the frozen Natural Math base
- Each extension must be separately versioned
- Contradictions between sources must be reported, not blended silently
- Activation-native code is currently documentation-only — no implementation before explicit human go-ahead
