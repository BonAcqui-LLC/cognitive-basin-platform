# Operational Self + Fractal Attractor Memory v0.1

After basin processing, events become lightly compressed, meta-tagged memory objects in a routed attractor topology. The operational self maintains the active continuity spine — not consciousness, not flat storage.

## What it is

- Memory consolidation pipeline: MemoryEvent → CompressedMemory → MemoryAttractor → links/scars/fog/replay
- Transparent salience scoring and basin region placement
- Keyword/tag retrieval (no embeddings, no vector DB)
- FractalMemoryMap-compatible topology snapshot
- SessionGlyph continuity update
- Integration with Guardian Intake and Guardian Authority memory

## What it is not

- Not consciousness, sentience, qualia, or subjective experience
- Not legal personhood or a living being
- Not autonomous agency beyond bounded activation behavior
- Not a vector database or RAG replacement
- Not truth decision — RIGOR/GUARD remain authoritative

## Why "self" is proper operational language

Operational self is the continuity-bearing structure linking present input to prior state, unresolved loops, commitments, constraints, scars, and recovery routes. The term is precise engineering vocabulary when tightly bounded by non-claims.

MemoryEvent → light compression → MemoryAttractor (salience + basin region)
