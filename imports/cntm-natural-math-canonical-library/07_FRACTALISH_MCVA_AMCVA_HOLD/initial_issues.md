# Initial Public Issues

These are the first issue titles and short descriptions prepared for the public `mcva` repository. They were not posted from this session because `BonAcqui-LLC/mcva` returned `404` through the GitHub connector at the time of release prep.

1. **Build release inventory**
   Create and maintain the publishability inventory for docs, code, datasets, and internal boundaries.

2. **Create MCVA Atlas manifest format**
   Define the canonical metadata schema for atlas entries and AMCVA boundary cases.

3. **Add box-counting fractal dimension descriptor**
   Add a descriptor implementation and tests for scale-bounded complexity estimation.

4. **Add lacunarity descriptor**
   Add a descriptor for gap structure and spatial heterogeneity.

5. **Add skeletonization and branch-count descriptor**
   Add a descriptor pass for topology-aware branching summaries.

6. **Add road crack public dataset demo**
   Add the first public-data application lane for roads and public works after rights verification.

7. **Add Natural Math synthetic trace generator improvements**
   Expand the generative side with better local memory, blockage, and branching behaviors.

8. **Add observed-vs-generated morphology distance scoring**
   Define a first comparison metric between observed morphology and generated candidate traces.

9. **Add SERA report output improvements**
   Improve metric reporting for waste, retries, false commitments, HOLD rate, and valid output.

10. **Define SymLan token criteria after descriptor stabilization**
    Document when descriptor clusters are stable enough to justify symbolic compression.

11. **Add pavement distress terminology mapping**
    Map morphology tokens to recognizable road-maintenance vocabulary.

12. **Add fracture/crack simulation lane**
    Add a public materials-failure lane that pairs descriptors with constrained growth or fracture traces.

13. **Add vascular/retinal public dataset research lane**
    Explore a cautious research-only biomedical lane with verified public data and explicit non-diagnostic framing.

14. **Add data license verification checklist**
    Create a required checklist before any third-party dataset or image enters the repo.

15. **Add whitepaper PDF export workflow**
    Add a reproducible path from Markdown whitepaper to shareable PDF.

16. **Add AI cognition / HOLD anti-drift research note**
    Capture future work around ternary unresolved states, provenance boundaries, and reasoning-cost SERA.

17. **Add Fractalish open-release website update**
    Mirror the open doctrine, non-claims, and GitHub pointers into the Fractalish public doorway.

18. **Add support / sponsorship page**
    Publish the honor-system sustainability path without turning the framework into closed licensing.

19. **Add bifurcation / recovery metric family**
    Add descriptor work for skeleton junctions, split angles, branch survival, reconvergence, post-split relaxation, and SVG annotation of bifurcation and recovery zones.

20. **Build Fractalish Commons MVP**
    Add the local analysis workbench, static Commons pages, SVG trace export, spreadsheet outputs, and starter reference library manifest.

21. **Quantify AMCVA subtypes**
    Estimate absence, obscuration, erasure, domain mismatch, geometric dominance, competing geometry, complementary geometry, and capture artifact instead of forcing a positive read.

22. **Add competing / complementary geometry scoring**
    Add transparent v0.1 estimates for branching, crack-network, circular/ring, spiral/vortex, grid/lattice, cellular/Voronoi, wave/periodicity, and artifact suspicion.
