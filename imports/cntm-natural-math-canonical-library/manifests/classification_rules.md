# Classification Rules

- `OPERATOR_CANON`: explicit operator-control document or current controlling prompt.
- `HISTORICAL_DISCOVERY`: preserve as-discovered artifact, even if later implementations are cleaner.
- `CURRENT_SPECIFICATION`: strongest located specification or public-facing design document in current use.
- `REFERENCE_IMPLEMENTATION`: runnable or package-based implementation used as current reference.
- `EXPERIMENTAL_IMPLEMENTATION`: exploratory implementation not yet the primary reference.
- `GENERATED_RESULT`: run output, frozen evidence, or derived artifact.
- `THEORY_CONTEXT`: supporting explanation, prompt lineage, or context artifact.
- `PHYSICAL_PLAN`: apparatus, engineering plan, or physical realization design.
- `PUBLICATION_DRAFT`: paper, appendix, or thesis draft intended for review/release.
- `SUPERSEDED`: older version carried forward only for lineage.
- `DUPLICATE`: exact duplicate material.
- `CONFLICTING`: materially different versions in the same title/version family.
- `UNCLASSIFIED_RELEVANT`: relevant but insufficiently resolved.
